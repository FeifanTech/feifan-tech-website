模块一：认识 RPA——什么是「数字机器人员工」？
用你最熟悉的场景理解 RPA
💡
一句话理解 RPA
RPA（Robotic Process Automation）就像雇了一个不知疲倦、不会出错的数字员工。 你在电脑上怎么操作，教它一遍，它就能自动重复执行——无论是打开网页、复制粘贴数据， 还是填写表单、发送邮件，全都可以自动化。

📊 做同一件事，人工 vs RPA 机器人

😓 人工处理（每月底）

  • ⏰登录系统，手动导出数据：20 分钟
  • 📋复制数据到 Excel：30 分钟
  • 🧮手动汇总计算：40 分钟
  • 📧手动发送给各部门：15 分钟
  • ⚠️偶尔出错，还要返工：30 分钟
总计：约 135 分钟 + 人工出错风险

🤖 RPA 机器人（配置完成后）

  • ✅自动登录、导出数据：2 分钟
  • ✅自动整理数据到 Excel：1 分钟
  • ✅自动计算汇总：30 秒
  • ✅自动发送给各部门：30 秒
  • ✅零出错，每次一致
总计：约 4 分钟，零错误，可定时执行

模块二：核心概念图解——用生活类比记住它们
5 个概念，理解 RPA 的工作方式
📋

流程 (Flow / 机器人)

自动化任务的完整程序
类比→📖 操作说明书

就像一份「新员工入职操作手册」，写清楚了「先做什么、再做什么、遇到问题怎么处理」。流程就是机器人的完整工作指南，从头到尾执行一遍就完成了任务。

🖱

动作 (Action)

流程中的单个操作步骤
类比→✋ 手的一个动作

就像人的每个手部动作：「点击鼠标」「按下键盘」「复制文字」「打开窗口」。每一步操作都是一个「动作」，影刀的动作库里有几百种预设动作，可以直接拖拽使用。

📝

变量 (Variable)

存储中间数据的容器
类比→🗒 便利贴

就像在办公桌上贴一张便利贴：「客户姓名：张三」。机器人执行时，把「从系统里读到的订单编号」存到变量里，下一步填写表单时，再从变量里取出来填进去。

🎯

元素选择器 (Selector)

定位网页/软件上的具体控件
类比→📍 精确的GPS地址

告诉机器人「要点击哪里」。就像导航软件需要精确地址才能定位，选择器告诉机器人「第 3 行、第 2 列的那个按钮」——点错地方会出错，所以选择器要精确。

⏰

触发器 (Trigger)

启动流程的条件
类比→⏰ 定时闹钟

就像设定闹钟「每天早上 8 点响」，触发器让机器人在指定时间或条件满足时自动启动流程，无需人工干预。例如：「每周一 9:00 自动运行报表流程」。

🔄

循环 (Loop)

重复执行相同动作
类比→🏭 流水线

就像流水线上，每个产品经过同样的处理步骤。「对 Excel 表格中每一行数据」循环执行「填写表单」这个动作，1000 行数据，机器人一条一条地自动处理完。

模块四：常用动作详解——你的工具箱
掌握这 4 类动作，可以完成 90% 的 RPA 任务

🌐 一、网页操作（最常用）

🖱

点击元素

点击网页上的任意按钮、链接、下拉菜单。用录制器捕获目标元素即可。

⌨

输入文字

在输入框中填写内容，可以输入固定文字或变量值（如 {{row.姓名}}）。

📋

获取文本

抓取网页上某个元素的文字内容，存入变量，供后续步骤使用。

📷

截图

自动截取当前屏幕或指定区域，可用于存档、验证操作结果或报告附件。

🔍

等待元素出现

等待某个元素加载完成后再执行下一步，避免网页加载慢导致点击失败。

📜

滚动页面

模拟鼠标滚动，用于触发懒加载内容或定位到页面底部的元素。

📊 二、Excel 操作（数据处理必备）

动作用途常见用法
打开工作簿打开指定路径的 Excel 文件路径可用变量动态指定
读取工作表把整张表读入变量（数据列表）搭配循环遍历每一行
获取单元格值读取特定行列的单元格读取表头/动态行数
写入单元格向指定单元格写入数据把抓取结果写回 Excel
新增行在表格末尾追加一行数据持续记录日志数据
保存工作簿保存并关闭文件操作完必须执行此步

📧 三、邮件操作

📨 发送邮件

配置收件人（可用变量）、主题、正文（支持 HTML 格式化）、附件（可动态绑定文件路径）。 适合定时发送报表、自动通知、异常告警。

📥 读取邮件（监控）

定时检查收件箱，当收到特定主题的邮件时自动触发处理流程。 例如：客户发送订单邮件 → 机器人自动解析附件 → 录入系统。

🔧 四、逻辑控制（让流程更智能）

🔀

条件判断（If/Else）

根据变量值决定走哪条路。例如：金额>1万 → 走审批流程，否则 → 直接录入。

🔄

循环（For Each）

对列表中的每个元素执行相同操作。处理 Excel 表格每一行必用。

🛡

异常处理（Try/Catch）

当某步骤出错时，不崩溃，转而执行备用方案或记录日志继续。

模块五：录制你的第一个自动化流程（10 分钟）
目标：自动在百度搜索，把结果截图保存到桌面
🎯
为什么从这个开始？
这个小案例覆盖了 RPA 最核心的三个能力：打开浏览器、输入内容、截图保存。 掌握这三步，就能举一反三，自动化大多数网页操作场景。
1

新建流程

打开影刀 → 点击「新建流程」→ 填写名称「搜索自动化-练习」→ 确认创建，进入流程编辑界面。

2

使用录制器（推荐新手）

点击顶部「录制」按钮 → 影刀最小化 → 你正常在电脑上操作：打开 Chrome、访问 baidu.com、在搜索框输入「飞凡科技」、点击搜索 → 回到影刀点「停止录制」。影刀会自动生成对应动作。

💡
录制时操作要慢一点，让影刀有时间捕获每个元素。快速操作可能会漏掉某些步骤。
3

手动添加截图动作

在动作库中搜索「截图」→ 拖拽「截取屏幕截图」动作到流程末尾 → 在属性面板配置保存路径（如 C:\Users\你的用户名\Desktop\搜索结果.png）。

4

运行测试！

点击顶部「▶ 运行」→ 观察机器人自动操作 → 检查桌面是否生成了截图文件。如果成功，恭喜你完成了第一个 RPA 流程！🎉

5

改造：把搜索词变成变量

点击「开始」节点 → 添加输入参数 keyword（搜索关键词） → 找到「输入文字」那步动作 → 把固定的「飞凡科技」改成 {{keyword}} → 再次运行时输入不同关键词测试。

📋 关键配置要点

📌 读取 Excel 数据

动作：「读取工作表」→ 选择文件路径和 Sheet 名称 → 勾选「包含表头」→ 结果存入变量 data_list

循环设置：拖入「For Each」→ 遍历变量 data_list → 循环变量命名为 row（代表每一行）

📌 填写表单时引用数据

在每个「输入文字」动作中，内容填写 {{row.列名}}，例如：

  • 姓名字段：{{row.姓名}}
  • 金额字段：{{row.金额}}
  • 日期字段：{{row.日期}}
⚠️
重要：加异常处理防止中断
在循环体内加「Try/Catch」异常处理：当某一行填写出错时（如网络超时）， 把该行的状态写为「失败」记录到 Excel，然后继续处理下一行， 而不是整个流程中断崩溃。这样 100 条数据里就算几条失败，其余的还是能完成。

🔑 核心配置：自动发邮件

发送邮件动作 — 配置示例 
动作：发送邮件（SMTP）

配置项：
  收件人：manager@company.com; boss@company.com
  主题：【自动报告】{{today}} 日销售数据汇总
  正文（HTML）：
    <h3>🤖 此邮件由 RPA 机器人自动生成</h3>
    <p>报告日期：{{today}}</p>
    <p>总销售额：{{total_amount}} 元</p>
    <p>订单数量：{{order_count}} 笔</p>
    <p>详细数据见附件。</p>
  附件：{{excel_file_path}}

变量说明：
  today        = 当前日期字符串（格式：2024-01-15）
  total_amount = 从网页抓取的总销售额
  order_count  = 从网页抓取的订单数量
  excel_file_path = 生成的 Excel 报表文件路径

⏰ 设置定时触发

1

在影刀中配置定时任务

流程列表页 → 找到你的流程 → 点击「...」更多 → 「定时设置」→ 选择「每天」→ 设置时间「08:00」→ 确认。

2

保持机器在线

定时任务在配置了任务的这台电脑上运行，需要确保电脑在触发时间处于开机且影刀在运行状态。如需长期无人值守，建议使用影刀云或专用的 RPA 服务器。

✅ 升级版：加入异常通知

在流程最外层加 Try/Catch，当任何步骤出错时，发一封异常告警邮件给运维人员，邮件包含错误截图和错误信息，方便快速定位问题。

🔗 进阶：与 Dify 结合

RPA 抓取原始数据后，通过 HTTP 请求调用 Dify 的 API，让 AI 对数据进行智能分析和洞察总结，再把 AI 生成的分析报告也附在邮件里。

进阶技巧 & 常见问题
少走弯路的实战经验

🎯 选择器稳定性技巧

  • 优先用 ID 属性定位元素（最稳定）
  • 避免用元素位置（第几行第几列）定位，网页改版就失效
  • 选择器编辑时，勾选「自动等待元素」
  • 对于动态内容，用「包含文字」方式匹配而非精确路径

🐛 调试技巧

  • 在关键步骤后加「截图」动作，方便查看机器人执行状态
  • 用「日志输出」动作打印变量值，确认数据是否正确读取
  • 先在少量数据（如 3 条）上测试，确认正确后再跑全量
  • 「单步执行」模式可以让流程一步一步走，便于定位问题

⚡ 提升稳定性

  • 关键操作前后加适当的「等待」动作（0.5-2秒），防止网页未加载
  • 所有网页操作都包裹在 Try/Catch 中
  • 操作结果写入日志文件，方便事后核查
  • 账号密码用「安全变量」存储，不要明文写在流程中

⚠️ 常见报错及解决

  • 「找不到元素」：网页加载慢，在操作前加「等待元素出现」动作
  • 「坐标点击偏移」：改为用选择器点击，而非坐标点击
  • 「Excel 文件锁定」：确保流程结束时执行「关闭工作簿」动作
  • 「验证码阻挡」：联系系统管理员开白名单，或使用 API 接口替代网页操作
