💡
一句话理解 Dify
Dify 是一个 AI 应用的积木工厂。就像乐高积木，它把「大语言模型」「知识库检索」「API 调用」「条件判断」等功能做成了标准积木块（节点）， 你只需要在画布上拖拽拼接，就能搭出自己想要的 AI 应用。

🤔 没有 Dify 之前，我们怎么做？

方式传统写代码用 Dify
门槛需要懂 Python、API 调用、后端部署拖拽操作，零代码
开发时间一个简单问答系统 1-2 周1-2 小时完成
调试改代码 → 重启 → 测试，反复折腾可视化调试，实时查看每步输出
修改需求改代码，可能影响其他功能直接在画布上调整节点
知识库需要自己搭 RAG 系统内置知识库，上传即用
发布需要服务器部署一键生成 API / 分享链接

🍕 餐厅类比：Dify 的三种工作模式

💬

聊天助手模式

就像一个服务员，你说什么它回什么。适合做对话机器人、客服助手、问答系统。

🔀

工作流模式

就像餐厅的后厨流程：点单→备料→烹饪→出餐。适合做多步骤自动化、批处理任务。

🕵️

Agent 模式

就像一个会自主规划的大厨，能根据情况自己决定用哪些工具。适合复杂推理任务。

🧩
模块二：核心概念图解——用生活类比记住它们
每个概念都有一个你熟悉的对应物
🔀

工作流 (Workflow)

整个自动化流程
类比→🍜 菜谱

就像一份菜谱，规定了从「准备食材」到「端上桌」的每一个步骤。工作流就是你整个 AI 应用的「流程图」，定义了数据从输入到输出经历哪些处理。

⬜

节点 (Node)

工作流中的单个处理步骤
类比→⚙️ 工厂设备

就像流水线上的一台机器，每个节点负责做一件事：有的节点负责「调用 AI」，有的负责「查知识库」，有的负责「根据条件分叉」。节点和节点之间用线连接。

📚

知识库 (Knowledge Base)

RAG 检索增强生成
类比→📖 公司图书馆

把你的企业文档、产品手册、FAQ 上传进去，AI 在回答问题时会先去「图书馆」查资料，再给出基于真实内容的回答，而不是「凭空想象」。

📦

变量 (Variable)

在节点间传递的数据
类比→📋 快递单号

就像快递单号，快递从一个城市运到另一个城市，单号始终跟着包裹走。变量就是数据在不同节点之间传递的「标签」，前一个节点的输出，可以作为下一个节点的输入。

🧠

LLM 节点

大语言模型调用
类比→👨‍🍳 大厨

这是整个工作流的「大脑」，负责理解、推理、生成文本。你告诉它「系统角色」（它扮演什么身份），给它「用户输入」，它就会按照你的要求生成回答。

🔌

HTTP 节点

调用外部接口
类比→📞 打电话给快递员

当你需要从外部系统获取数据（查天气、查数据库、发通知）时，HTTP 节点就像打一个电话：你发出请求，等对方接听并告知结果，再把结果传给下一步。

模块四：常用节点详解——认识你的积木
掌握这 7 个节点，可以搭出 90% 的 AI 应用
▶

开始节点

每个工作流的起点，定义用户的输入变量（如：用户问题、上传的文件）。

🧠

LLM 节点

调用大语言模型（GPT、Claude 等），配置系统提示词，输入内容，获取 AI 生成的结果。

🔍

知识检索节点

从你上传的知识库中检索相关内容，把检索结果传给 LLM，实现基于真实文档的回答。

⚖

条件分支节点

像代码里的 if-else。根据某个变量的值，决定走哪条路。例如：用户评分>4 → 感谢，否则 → 安抚。

🔌

HTTP 请求节点

向任意外部 API 发请求。比如：查询天气、调用企业数据库、向钉钉发通知。

💻

代码节点

当节点不够用时，可以写几行 Python 代码做数据处理，比如格式化日期、计算数值。

🔄

循环节点

对一个列表中的每个元素重复执行相同操作。例如：批量分析 10 份合同，每份合同走一遍流程。

📝

变量赋值节点

在工作流中途修改或合并变量的值，为后续节点做数据整合。

⏹

结束节点

工作流的终点，定义最终输出给用户的内容（直接输出文本或引用某个变量）。

🔗 节点连接规则

节点 A
输出：{{result}}
────→
节点 B
输入：{{result}}
A 的输出 = B 的输入

⚡ 重要：连接线不只是视觉上的箭头，它代表「数据传递」。节点 A 产生的变量，可以在节点 B 的配置中用 {{变量名}} 引用。

⚡
模块五：搭建你的第一个工作流（15 分钟）
目标：搭一个「文本摘要生成器」，输入长文章，输出摘要
🎯
完成标准
在 Dify 画布上，能看到 开始 → LLM → 结束 三个节点连接好，输入一段文字后，能输出 AI 生成的摘要。
1

新建工作流应用

登录 Dify → 点击「创建应用」→ 选择「工作流」类型 → 填写名称「文章摘要生成器」→ 点击确认。

2

配置「开始节点」——定义输入

点击画布上已有的「开始」节点 → 右侧面板中点击「+添加变量」→ 变量名填 article，类型选「文本」，显示名称填「文章内容」→ 保存。

3

添加 LLM 节点

从左侧节点库拖拽「LLM」节点到画布 → 用鼠标从「开始」节点右侧的连接点拖出一条线，连接到 LLM 节点左侧。

4

配置 LLM 节点——写提示词

点击 LLM 节点 → 右侧面板选择模型（如 gpt-4o-mini）→ 在「系统提示词」填写：

系统提示词 (System Prompt) 
你是一个专业的文章摘要助手。请将用户提供的文章，提炼成一份简洁、准确的中文摘要，要求：
1. 摘要控制在 200 字以内
2. 保留核心观点和关键数据
3. 使用清晰的分段展示

在「用户消息」框中，点击「插入变量」→ 选择 article（就是开始节点定义的输入变量）。

5

连接结束节点

画布上已有「结束」节点 → 将 LLM 节点连接到结束节点 → 点击结束节点，在右侧「输出内容」选择 LLM 节点的输出变量。

6

测试运行！

点击右上角「▶ 运行」→ 在弹出的输入框中粘贴一段文章 → 点击「运行」→ 查看每个节点的输入/输出和最终结果。🎉

💡 小技巧：按住 Ctrl+Z 可以撤销，不用担心操作失误

实战项目①：智能客服机器人（含知识库）
目标：搭建一个能回答产品问题、不乱编的客服 AI
⚠️
为什么要用知识库？
直接用 LLM 回答客户问题，AI 可能会「幻觉」——编出不存在的产品信息。 加入知识库后，AI 先查你上传的文档，再基于真实内容作答，避免出错。

📋 分步构建指南

1

创建知识库

进入 Dify 左侧菜单「知识库」→「创建知识库」→ 上传你的产品文档（PDF/Word/TXT 均可）→ 选择「自动分段」→ 等待索引完成（通常 1-5 分钟）。

💡
建议把文档整理清楚再上传：一个知识库只放同类文档（如：产品手册、FAQ、价格表），这样检索更准确。
2

新建聊天助手应用

「创建应用」→ 选择「聊天助手」→ 在「编排」页面，点击「上下文」→「添加」→ 选择你刚才创建的知识库。

3

写系统提示词

系统提示词模板 
你是【公司名称】的专业客服助手，名字叫「小飞」。

【行为准则】
1. 只基于提供的知识库内容回答问题，不要编造信息
2. 如果知识库中找不到相关内容，请如实说「这个问题我需要帮你转接人工客服确认」
3. 回答要简洁、友好、专业
4. 如果用户的问题不清楚，先确认再回答

【上下文】
{{#context#}}（这里会自动填入从知识库检索到的相关内容）
4

调整检索设置

点击知识库右侧的「设置」→ 检索方式选「混合检索」（语义+关键词，效果最好）→ TopK 设置为 3-5（检索最相关的 3-5 段）→ 相似度阈值设 0.5。

5

测试 & 发布

在右侧预览框中测试几个典型问题 → 如果回答不准确，检查知识库内容是否完整 → 满意后点「发布」→ 可以获得分享链接或 API 地址。

🎉
恭喜！你的第一个 RAG 客服已完成
现在，无论用户问什么，AI 都会先去你的文档里找答案，不会乱编。 你可以把分享链接发给团队测试，收集反馈后继续优化知识库内容。

合同文本：


进阶技巧 & 常见问题
用好这些技巧，效率提升 3 倍

🐛 调试技巧

  • 单节点测试：点击节点右上角「▶」，单独测试该节点不用全流程跑
  • 查看中间变量：运行后点击节点，可看到每步的实际输入/输出
  • 开启「记录日志」：在工作流设置中开启，自动保存每次运行记录

🎛 提示词优化

  • 给 AI 一个明确角色：「你是...」比「请帮我...」效果好
  • 指定输出格式：「以 JSON 格式输出」「用 Markdown 标题分段」
  • 给例子：「例如：输入 A，输出 B」效果最稳定
  • 限制范围：「只基于以下内容回答，不要添加额外信息」

🚀 发布 & API 集成

  • 发布为网页：点「发布」→「运行」，获得可分享的聊天链接
  • 获取 API：在「访问 API」页面，获取 API Key 和调用示例
  • 嵌入网页：Dify 提供 iframe 代码，可直接嵌入任何网页
  • Webhook 触发：配置 HTTP 节点监听，实现事件驱动自动化

⚠️ 常见报错及解决

  • 「变量未定义」：检查前一个节点是否已定义该变量，名称是否拼写一致
  • 「知识库无结果」：降低相似度阈值，或检查文档内容是否已成功索引
  • 「超出 Token 限制」：在 LLM 节点设置最大输入长度，或先用代码节点截断
  • 「API 超时」：HTTP 节点增加超时时间，或添加重试逻辑
