🔌 一句话理解 MCP

MCP = AI 的 USB 接口。任何系统，只要实现了 MCP Server，AI 就能操作它。

MCP（Model Context Protocol）是 Anthropic 提出的开放标准协议，让 AI 通过标准接口连接任意外部系统——就像 USB 接口让你可以连接任何外设一样。

MCP Server 开发门槛（工程师视角）
✅
核心结论：MCP Server 开发成本约等于写一组 REST API

对工程师而言，把自建系统接入 AI 生态是完全可行的工程任务。以下是一个最简单的 Python MCP Server 示例（约 50 行代码）：

# 最简单的 MCP Server 示例：把公司项目查询接口暴露给 AI # 安装：pip install mcp from mcp.server import Server from mcp.server.stdio import stdio_server from mcp.types import Tool, TextContent import json server = Server("company-tools") # 定义 AI 可以调用的工具（函数） @server.list_tools() async def list_tools(): return [ Tool( name="get_project_status", description="查询指定项目的当前状态、里程碑和风险", inputSchema={ "type": "object", "properties": { "project_id": {"type": "string", "description": "项目 ID"} } } ) ] # 实现工具的具体逻辑 @server.call_tool() async def call_tool(name: str, arguments: dict): if name == "get_project_status": # 调用你公司的内部 API data = fetch_from_internal_api(arguments["project_id"]) return [TextContent(type="text", text=json.dumps(data, ensure_ascii=False))] # 启动 Server（Claude Desktop / Cursor 等通过 stdio 调用） if __name__ == "__main__": import asyncio asyncio.run(stdio_server(main(server)))

模块 2｜AI Native 工作方式——思维的最终转变
25 分钟 · 自我诊断 + 认知升级
三个阶段的自我诊断
阶段 1：AI 辅助（Tool User）
  • 偶尔用 AI 处理特定任务
  • 大部分时候靠自己
  • AI 是"锦上添花"的工具
  • 遇到问题才想到用 AI
大多数人现在所在的位置
阶段 2：AI 优先（AI First）
  • 遇到任务先问"AI 能做哪些"
  • 习惯用 AI 处理 A 类任务
  • 自己专注 B/C 类判断
  • 有个人提示词库
本课程的培训目标
🏆 阶段 3：AI Native
  • 从设计阶段就考虑 AI 介入点
  • 团队协作方式围绕 AI 能力重构
  • AI 是基础设施而不是工具
  • Skills 持续迭代形成竞争壁垒
长期发展目标
AI Native 的四个具体特征
📄

文档即 Prompt 材料

写需求文档时，同步考虑"这份文档能直接喂给 AI 处理吗"。结构化程度更高，为 AI 可处理性而设计。

📦

输出物标准化

方案、代码、测试用例都有 AI 可处理的模板。知识沉淀到知识库而不是个人脑袋，离职不带走价值。

⚖️

人机边界清晰

团队明确哪些角色的哪些工作由 AI 完成，人专注更高阶的判断和关系维护。边界模糊是最大效率杀手。

🔄

Skills 持续迭代

把团队沉淀的最佳实践持续转化为可复用的 Skills。这是可积累的智力资产，形成竞争壁垒。

AI 时代的个人竞争力重构
⚠️ 被 AI 替代的能力（低附加值）
  • 重复性文档整理
  • 样板代码编写
  • 基础数据分析报告
  • 格式转换处理
  • 标准化文案撰写
🚀 被 AI 放大的能力（高杠杆）
  • 执行速度（AI 加速执行）
  • 代码产量（3-10x 提升）
  • 方案覆盖度（更全面）
  • 文档质量（更结构化）
  • 响应速度（更及时）
💡
AI 替代不了的核心能力
  • 复杂判断与决策：需要综合多方信息、价值观、利益权衡的决策
  • 利益协调：理解各方诉求，找到多方共赢的方案
  • 客户信任建立：长期积累的人际关系和信任
  • 创意方向把控：对"什么是好的"的审美判断和方向感

结论：未来竞争不是"你 vs AI"，而是"会用 AI 的你 vs 不会用 AI 的你的竞争对手"

模块 3｜专题课：AI 辅助售前方案全流程
50 分钟 · 完整实操，现场生成一份售前方案初稿
🎯
本模块目标

从客户需求输入，到可交付的售前方案，全程 AI 辅助，人只做方向判断和最终把关。现场走完整个流程，每位学员当天带走一份可用的方案初稿。

1

客户需求消化

⏱️ 约 10 分钟
📥 输入：客户需求文档 / 会议纪要 / 邮件（脱敏真实案例）
📤 产出：结构化需求分析报告（2 页内）

让 AI 完成：① 提炼客户核心痛点（按优先级排序）；② 识别客户未说出口的隐性需求；③ 识别需求中的技术风险和边界模糊点；④ 生成需要向客户确认的问题清单。

你是一位有 10 年经验的解决方案架构师（SA）。
请仔细阅读以下客户需求材料，完成四个任务：

1. 【痛点分析】提炼客户核心痛点（按重要程度排序，≤5条）
2. 【隐性需求】识别客户没有明说但可能存在的深层需求
3. 【技术风险】列出需求中的技术风险和边界模糊点
4. 【确认清单】生成需要向客户确认的问题列表（可直接发给客户）

输出格式：Markdown，分四个章节，每章节 5-8 条要点。

---材料开始---
[粘贴脱敏的客户需求文档]
---材料结束---
✅ 人工审核：确认痛点优先级是否准确
2

方案框架设计

⏱️ 约 10 分钟
📥 输入：Step 1 的需求分析 + 公司产品能力说明（从 Dify 知识库 RAG 检索）
📤 产出：方案大纲（可与讲师/同事快速对齐）

让 AI 生成方案目录结构，评估每个章节的论述重点，并建议差异化竞争角度（为什么选我们而不是竞争对手）。

基于以上需求分析，以及我们公司的产品能力：
[从 Dify 知识库粘贴：公司产品能力说明]

请生成一份售前方案的目录结构，包含：
1. 方案章节列表（背景/痛点/解决方案/技术架构/实施计划/预期价值/报价逻辑）
2. 每章节的 2-3 个核心论述重点
3. 建议 2-3 个差异化竞争角度（对比主要竞争对手）

输出格式：Markdown，结构清晰
✅ 人工审核：确认方案方向和竞争角度
3

核心内容生成

⏱️ 约 15 分钟
📥 输入：大纲各章节 + 技术架构说明 + 历史案例（RAG 检索）
📤 产出：方案各章节初稿

按大纲分章节，逐块让 AI 生成初稿。重点演示三个场景：

📐 技术架构描述

给出架构图说明，AI 生成专业的文字解释，包含关键技术选型和设计决策依据。

📅 实施计划生成

输入项目范围和团队规模，AI 生成分阶段实施计划（含里程碑和风险说明）。

💰 ROI 价值主张

输入客户痛点和方案能力，AI 生成量化的预期价值陈述（ROI 框架），增强说服力。

【实施计划生成示例 Prompt】
你是一位有 10 年经验的项目经理（PMP）。
基于以下项目信息，生成分阶段实施计划：

项目范围：[填入]
团队规模：[填入]
时间约束：[填入]
关键里程碑要求：[填入]

请生成：
1. 分阶段计划（Phase 1/2/3 及其目标和时间范围）
2. 每阶段关键里程碑（Milestone）
3. 主要风险及应对措施
4. 对客户的资源投入要求

格式：表格 + 甘特图（文字版）
4

PPT 生成与优化

⏱️ 约 10 分钟
📥 输入：完整方案文字稿
📤 产出：PPT 大纲 + 每页关键信息点

用 AI（配合飞书 / Gamma / ChatGPT）把文字方案转化为 PPT 大纲。用提示词控制结构、风格和信息密度。

请将以下方案文字稿转化为 PPT 演示大纲：

【规格要求】
- 总页数：15-20 页
- 风格：商务专业，信息密度中等
- 每页：1 个核心观点 + 2-3 个支撑要点
- 特别要求：每页给出建议的视觉元素（图表/流程图/数字大字/对比表格）

【方案文字稿】
[粘贴方案文字内容]
⚠️
边界认知：AI 生成结构和文字，设计排版还需要人工或专业工具（Gamma / 飞书 / PowerPoint）完成最终美化。

⏱️ 约 5 分钟

  • 报价策略中的商务判断
