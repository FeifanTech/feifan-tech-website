🔌 接口与协议层

接口与协议
🔧 Tools Use
AI 世界 → 软件工程
SDK / 函数调用 (Function Call)
AI 模型调用外部工具（搜索、计算、API），就像程序调用 SDK 中的函数。 模型决定"调哪个函数、传什么参数"，运行时负责执行并返回结果。
接口与协议
🔗 MCP
AI 世界 → 软件工程
REST API / OpenAPI 规范
Model Context Protocol 是让 AI 连接外部系统的标准接口协议。 类比 REST / OpenAPI：定义了请求格式、端点、认证方式，让不同服务可以互联互通。
接口与协议
🧩 Skills
AI 世界 → 软件工程
插件 / Plugin
Skills 是 AI 平台上可插拔的能力模块（如"翻译"、"代码审查"）， 相当于 IDE 插件或 WordPress 插件——即装即用，按需组合。

⚙️ 执行与运行层

执行与运行
🏗️ Harness
AI 世界 → 软件工程
运行时框架 / Runtime Framework
Harness 是承载 AI Agent 运行的底层框架（调度、编排、生命周期管理）， 类似 Spring Boot / Express.js 之于 Web 服务——提供脚手架，让开发者专注业务。
执行与运行
🖥️ Computer Use
AI 世界 → 软件工程
RPA / 桌面自动化
AI 直接操控鼠标键盘来使用桌面应用程序，就像 RPA（机器人流程自动化）工具一样—— 看着屏幕、点击按钮、输入数据，模拟人类操作。
执行与运行
🌐 Browse Use
AI 世界 → 软件工程
Selenium / Playwright 浏览器自动化
AI 自动打开浏览器、浏览网页、点击链接、提取内容， 就像 QA 工程师用 Selenium 写的自动化测试脚本，只不过"脚本"由 AI 实时决策生成。
执行与运行
🤖 Agent
AI 世界 → 软件工程
微服务 / 守护进程 (Daemon)
Agent 是能自主规划、执行多步任务的 AI 实体。 类似微服务架构中的自治服务——有自己的职责边界，能独立决策，通过消息与其他组件协作。
执行与运行
🔄 Workflow
AI 世界 → 软件工程
CI/CD Pipeline / 流水线
AI Workflow 将多个步骤（提取→分析→生成→审核）串联成自动化流程。 就像 Jenkins / GitHub Actions 的 Pipeline——定义阶段、条件分支、自动触发。

📊 数据与知识层

数据与知识
📚 RAG
AI 世界 → 软件工程
数据库查询 + 模板引擎
Retrieval-Augmented Generation = 先检索（SELECT）相关资料，再基于资料生成回答。 类比：先查数据库拿到数据，再用模板引擎（如 Mustache）渲染成最终页面。
数据与知识
🧬 Embedding
AI 世界 → 软件工程
索引 / 倒排索引 (Inverted Index)
Embedding 将文本转化为高维向量，让语义相似的内容靠近。 类似 Elasticsearch 的倒排索引——目的都是让"搜索"变得快速且准确。
数据与知识
📦 Context Window
AI 世界 → 软件工程
工作内存 / RAM
模型一次对话能"看到"的信息上限。超出窗口的内容就会"遗忘"。 就像程序的可用内存——内存不够就需要分页加载或外部存储。
数据与知识
🔢 Token
AI 世界 → 软件工程
字节 (Byte) / 数据单元
Token 是 AI 模型处理文本的最小单位（大约 ¾ 个英文单词）。 类比字节是计算机存储的基本单位——计费、限制都按这个来算。

🛡️ 质量与可靠性

质量与可靠性
👻 Hallucination
AI 世界 → 软件工程
Bug / 数据错误
AI "一本正经地胡说八道"，生成看似合理但事实错误的内容。 就像软件 Bug——程序运行了、有输出了，但结果是错的。需要测试和验证机制来防范。
质量与可靠性
🧠 Chain of Thought
AI 世界 → 软件工程
分步调试 / Debug Trace
让 AI "展示思考过程"，一步步推理而非直接给答案。 就像开发者用 debugger 逐行执行、打印变量——过程可见，更容易定位问题。

🏛️ 架构与工程化

架构与工程化
✍️ Prompt Engineering
AI 世界 → 软件工程
需求规格说明书 / SRS
精心设计 Prompt 就像写 SRS（软件需求规格说明）—— 你描述得越清晰、越结构化，AI 交付的结果就越接近预期。"Garbage in, garbage out" 同样适用。
架构与工程化
🔨 Fine-tuning
AI 世界 → 软件工程
二次开发 / 定制化开发
在通用模型基础上用专属数据再训练，使其适配特定场景。 类似在开源框架上做二次开发——底层能力不变，上层行为按需定制。

类比核心洞察

软件工程的核心概念——接口标准化（MCP ↔ REST）、模块化（Skills ↔ Plugin）、 流程自动化（Workflow ↔ CI/CD）——在 AI 时代并没有消失，只是换了一种表达形式。 掌握软件工程基本功的人，理解 AI 新概念会快得多。
