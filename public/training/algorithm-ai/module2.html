5个核心维度，系统化评估每个算法方案
📐
算法选择没有银弹。同一个问题，不同的业务场景、资源约束和数据情况，最优算法可能截然不同。5维度框架帮助你在这些因素中做出有理有据的权衡。
🗂️

维度① 问题类型

首先确定问题属于哪类ML任务：监督/无监督/强化学习？回归/分类/推荐/时序/CV？不同问题类型对应完全不同的算法家族。

💡 关键问题：目标变量是连续值、离散类别还是排序？
📊

维度② 数据规模

样本量决定可选算法范围。<1万样本适合传统ML；1万~100万样本传统ML和轻量深度学习均可；>100万样本深度学习优势开始显现。特征维度高时需要注意维度诅咒。

💡 经验法则：神经网络需要的数据量通常是传统ML的10倍以上
🎯

维度③ 精度要求

高精度需求（如医疗诊断、金融风控）优先考虑集成方法或深度学习；中等精度场景可接受简单模型换取可维护性；精度与其他维度需要权衡，不要一味追求精度。

💡 精度提升10%，工程复杂度可能增加100%
⚡

维度④ 延迟约束

在线推理延迟要求直接限制模型复杂度。实时场景（<10ms）只能用简单模型或预计算；近实时（10ms~100ms）可用轻量级神经网络；离线批量无延迟约束，可用复杂模型。

💡 深度学习模型通常需要GPU才能达到实时推理要求
🔍

维度⑤ 可解释性

业务方需要理解模型决策依据时，优先选择可解释模型（线性模型、决策树）。使用黑盒模型时，需配合SHAP/LIME等解释工具。金融、医疗领域通常有强制性的可解释性要求。

💡 SHAP可以为任何树模型提供特征贡献度解释

决策树使用提示
决策树给出的是候选算法范围，不是最终答案。在候选范围内，还需要结合维度②~⑤（数据规模/精度/延迟/解释性）做进一步权衡。通常建议从最简单的可行方案开始实验。

传统ML vs 深度学习 vs 强化学习
三大技术路线的核心差异与适用边界
ML

传统机器学习

  • 数据量：千~百万级
  • 特征：需人工设计
  • 训练速度：快（分钟级）
  • 可解释性：强
  • 部署成本：低
  • 代表算法：XGBoost、LightGBM、随机森林、逻辑回归
DL

深度学习

  • 数据量：百万级以上
  • 特征：自动提取
  • 训练速度：慢（小时~天）
  • 可解释性：弱
  • 部署成本：高（需GPU）
  • 代表算法：BERT、ResNet、Transformer、LSTM
RL

强化学习

  • 数据：通过交互产生
  • 目标：最大化长期奖励
  • 训练周期：长
  • 可解释性：弱
  • 适用：序列决策、控制问题
  • 代表算法：DQN、PPO、SAC、A3C
对比维度 传统ML（XGBoost等） 深度学习 强化学习
最小数据量 1,000+ 样本 10,000+ 样本 依赖模拟器/环境
特征工程 需要大量人工设计 可端到端自动提取 状态空间设计
训练时间 分钟~小时 小时~天（GPU） 天~周
推理延迟 <1ms（CPU可用） 5ms~100ms（依赖GPU） <1ms（策略网络小）
可解释性 高（SHAP支持好） 低（需LIME/GradCAM） 低
过拟合风险 中（需正则化） 高（需大量数据/Dropout） 中（探索与利用平衡）
飞凡适用场景 销量预测、购买预测 商品推荐、水产视觉 IoT控制优化（未来）
⚠️
飞凡当前阶段建议
在算法团队建立初期，建议以传统ML（特别是LightGBM/XGBoost）为主力。这类算法工程成本低、效果稳定、可解释性好，适合快速验证业务价值。深度学习应在数据积累充足、业务价值充分验证后再逐步引入。

Python示例：使用sklearn进行快速算法对比

Python 多算法快速基准对比 
from sklearn.model_selection import cross_val_score, StratifiedKFold
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from sklearn.tree import DecisionTreeClassifier
import lightgbm as lgb
import pandas as pd
import numpy as np

# ── 准备候选算法列表 ──
candidates = {
    '逻辑回归 (基线)': LogisticRegression(max_iter=500),
    '决策树': DecisionTreeClassifier(max_depth=8),
    '随机森林': RandomForestClassifier(n_estimators=100),
    '梯度提升': GradientBoostingClassifier(),
    'LightGBM': lgb.LGBMClassifier(verbose=-1),
}

# ── 5折交叉验证评估 ──
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
results = {}

for name, model in candidates.items():
    scores = cross_val_score(model, X_train, y_train, cv=cv, scoring='roc_auc')
    results[name] = {
        'AUC均值': scores.mean(),
        'AUC标准差': scores.std(),
    }

# ── 输出对比报告 ──
result_df = pd.DataFrame(results).T.sort_values('AUC均值', ascending=False)
print(result_df.to_string(float_format='{:.4f}'.format))

# ── 根据结果做选型决策 ──
best_model = result_df.index[0]
best_auc = result_df.iloc[0]['AUC均值']
baseline_auc = results['逻辑回归 (基线)']['AUC均值']
improvement = (best_auc - baseline_auc) / baseline_auc * 100

print(f"\n最优算法: {best_model}, AUC={best_auc:.4f}")
print(f"相比基线提升: {improvement:.1f}%")
print(f"{'值得工程化部署' if improvement > 5 else '提升有限，考虑坚持使用基线'}")

飞凡业务场景 × 算法方案
5个核心场景的推荐算法路线与选型理由
📈

场景①：商家销量预测

时序预测 | 离线批量 | 需要可解释
🥇 推荐方案
Prophet + LightGBM 集成：Prophet捕捉趋势和节假日效应，LightGBM处理促销、天气等外生特征交互
🥈 备选方案
SARIMA（数据稀疏的SKU）、N-BEATS（数据充足且精度优先）
选型理由：可解释性强（趋势/节假日组件可分解），工程成本低，对中小数据量友好；新品用全局均值降级
🛍️

场景②：用户商品推荐

推荐 | 近实时 | 冷启动挑战
🥇 推荐方案
两阶段：向量召回 + LightGBM精排：Faiss向量检索做召回（快），LightGBM做精排（准）
🥈 备选方案
协同过滤（数据少时）、Wide&Deep（数据充足、特征丰富时）
冷启动策略：新用户用热门商品兜底，新商品用内容相似度召回（基于商品标签/类目）
🎯

场景③：购买意愿预测

二分类 | 批量离线 | 样本不均衡
🥇 推荐方案
LightGBM + 阈值优化：LightGBM输出购买概率，根据业务成本矩阵调整分类阈值
🥈 备选方案
XGBoost（效果相近）、逻辑回归（基线对比 + 可解释性审计）
不均衡处理：class_weight='balanced' + 过采样SMOTE（少数类）；评估指标用AUC-ROC和业务转化收益
🌡️

场景④：IoT智能环境控制

实时控制 | 边缘端 | 强约束
🥇 推荐方案
规则引擎 + 决策树分层：规则保障安全边界，决策树优化边界内的控制策略
📅 长期方案
PPO强化学习（数据积累6个月以上后，构建仿真环境训练）
选型理由：边缘端资源有限（无GPU），安全优先，规则+决策树可在嵌入式系统运行，延迟<5ms
🐟

场景⑤：水产养殖智能监控

目标检测 | 视频流 | 需要GPU
🥇 推荐方案
YOLOv8n（轻量版） + 行为追踪：YOLOv8检测鱼群位置，ByteTrack多目标追踪，异常行为规则触发告警
🥈 备选方案
EfficientDet（精度优先）、MobileNetV3（资源极度受限时）
数据策略：先标注200张高质量样本做迁移学习，再通过主动学习扩充标注；夜间红外图像需单独处理
