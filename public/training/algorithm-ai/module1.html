⚠️
最常见的错误：从算法出发
许多工程师在接到需求后，第一反应是"用神经网络来做"或"用随机森林吧"——这种从算法出发的思维方式，往往导致过度工程化、模型不可解释、落地困难等问题。正确做法是：从业务问题出发，再寻找最合适的算法。

业务场景分析方法论
三步框架：目标定义 → 约束识别 → 数据评估

三步业务分析框架

1

目标定义

明确业务目标是什么？成功的标准是什么？用数字可量化的指标表达。

2

约束识别

有哪些限制条件？响应时间、计算资源、可解释性需求、法规合规要求。

3

数据评估

有什么数据可用？数据质量如何？数据量是否足够？标注成本高不高？

步骤一：目标定义详解

1

业务目标转化为ML目标

业务目标通常是模糊的（"提升销量"），需要转化为可优化的ML目标（"预测未来7天销量，MAPE < 15%"）。

2

确定预测/决策对象

每个样本代表什么？是一个订单、一个用户、一个SKU、还是一个时间段？粒度决定特征工程策略。

3

确定评估指标

预测类用RMSE/MAE/MAPE，分类类用Precision/Recall/F1/AUC，推荐类用Hit Rate/NDCG。与业务方确认最重要的指标。

4

设定基线与目标值

当前方案（规则或人工）的表现是基线。算法方案需要在基线上提升多少才有落地价值？

步骤二：约束识别详解

约束类型 具体问题 对算法的影响
响应时间 需要实时预测还是离线批量？延迟上限是多少ms？ 实时<100ms → 避免复杂深度学习，优选轻量模型
计算资源 是否有GPU？内存限制？边缘设备还是云端？ 边缘端 → 模型量化/剪枝，优选决策树类
可解释性 业务方是否需要理解模型决策？是否有审计需求？ 高解释性需求 → 逻辑回归/决策树 over 深度学习
更新频率 模型需要多久更新一次？在线学习还是离线重训？ 高频更新 → 选择训练速度快的算法
数据合规 数据是否涉及隐私？是否需要联邦学习？ 隐私敏感 → 联邦学习/差分隐私或仅用聚合特征

步骤三：数据评估框架（5W1H）

数据5W1H评估清单
  • 📦 What：有哪些数据源？字段含义是什么？
  • 📊 How Much：样本量多大？历史多久？
  • 🎯 How Clean：缺失率？异常值比例？
  • ⏱ When：数据时效性？能否实时获取？
  • 🏷️ Label：有没有标签？标注成本如何？
  • ⚖️ Bias：数据分布是否有偏？样本不均衡？
🚨 数据风险红线
  • • 样本量 < 1000 → 优先规则/统计方法
  • • 标签缺失/噪声严重 → 数据清洗优先于建模
  • • 正负样本比 > 50:1 → 需要采样策略
  • • 历史数据不足6个月 → 时序模型无法训练
  • • 特征时序泄露 → 会导致线下好线上差

📌 实战演练：以"商家销量预测"为例

框架应用示例 — 商家销量预测
🎯 业务目标
帮助商家提前备货，减少缺货和积压，降低库存成本
📐 ML目标
预测未来7天、SKU级别的日销售量，MAPE目标 < 20%
📏 评估指标
主指标：MAPE；辅助指标：MAE、偏差方向（高估vs低估影响不同）
⏱ 响应约束
离线预测，每天凌晨3点批量运行，无实时延迟要求
💻 计算约束
可用2核4G服务器，无GPU，预测结果写入MySQL
🔍 可解释性
商家需要理解为什么预测偏高/偏低（节假日、促销影响）
📦 数据概况
2年历史销售记录，SKU粒度，含商品属性、促销标记、节假日标记
⚠️ 数据问题
新品（<30天历史）约占15%，部分SKU数据稀疏，大促期间数据分布漂移
✅ 算法方向
可解释时序模型（Prophet）+ 梯度提升（LightGBM）集成，分别处理趋势和特征交互
🗂️
算法问题分类体系
7类核心问题类型，覆盖飞凡所有业务场景
💡
正确识别问题类型是算法选择的第一步。同一业务问题可以从多个角度定义，不同的问题定义会导致截然不同的算法选型。
📈

回归（Regression）

预测连续数值目标，如销售量、价格、温度。
线性回归 · Ridge/Lasso · XGBoost · LightGBM · 神经网络
🏢 飞凡：商家销量预测、水温预测
🏷️

分类（Classification）

将输入归入预定义类别，如是否购买、是否异常。
逻辑回归 · 决策树 · XGBoost · LightGBM · SVM
🏢 飞凡：购买意愿预测、异常行为检测
🔵

聚类（Clustering）

无监督发现数据中的自然分组，无需预先标注。
K-Means · DBSCAN · 层次聚类 · 高斯混合模型
🏢 飞凡：用户分群、商品品类聚合
🛍️

推荐（Recommendation）

从大量候选中为用户匹配最相关的物品或内容。
协同过滤 · Wide&Deep · 矩阵分解 · 向量检索
🏢 飞凡：用户商品推荐系统
📅

时序预测（Time Series）

利用时间序列历史数据预测未来趋势和周期模式。
SARIMA · Prophet · N-BEATS · Temporal Fusion Transformer
🏢 飞凡：销量预测、设备状态趋势预测
🚨

异常检测（Anomaly Detection）

识别数据中的罕见事件或异常样本，通常正样本极少。
Isolation Forest · LOF · Autoencoder · 3σ规则
🏢 飞凡：水产异常行为识别、IoT传感器故障
👁️

计算机视觉（CV）

从图像或视频中提取信息，实现目标检测、分割、识别。
YOLOv8 · ResNet · EfficientNet · SAM · 迁移学习
🏢 飞凡：水产养殖鱼群监控、设备状态图像识别

常见误区 vs 正确做法

❌ 常见误区
  • 先选算法再找数据来支撑
  • 盲目追求最新最复杂的模型
  • 忽略业务约束（延迟/解释性）
  • 用准确率作为唯一评价指标
  • 把数据分析阶段跳过直接建模
  • 不考虑模型上线后的维护成本
✅ 正确做法
  • 先做业务分析，再确定算法方向
  • 从最简单的可行方案开始迭代
  • 在算法选型时就考虑部署约束
  • 与业务方共同定义有意义的指标
  • 用EDA充分理解数据再建模
  • 评估模型整个生命周期的成本

Python示例：业务数据分析模板

Python 业务数据初探模板 — 商家销量预测场景 
# ============================================================
# 业务数据分析模板 — 使用前请根据实际字段名修改
# ============================================================

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

# ── Step 1: 加载数据，了解基本情况 ──
df = pd.read_csv('sales_data.csv', parse_dates=['date'])
print("数据维度:", df.shape)
print(df.dtypes)
print(df.head())

# ── Step 2: 数据质量评估 ──
def data_quality_report(df):
    report = pd.DataFrame({
        '缺失值': df.isnull().sum(),
        '缺失率': (df.isnull().sum() / len(df) * 100).round(2),
        '唯一值': df.nunique(),
        '类型': df.dtypes
    })
    return report

print(data_quality_report(df))

# ── Step 3: 目标变量分析（销量分布）──
target_col = 'sales_qty'
print(f"销量基本统计:\n", df[target_col].describe())
print(f"零销量比例: {(df[target_col]==0).mean():.2%}")
print(f"偏度: {df[target_col].skew():.3f}")  # 偏度>1说明需要对数变换

# ── Step 4: 时序特征分析 ──
df['date'] = pd.to_datetime(df['date'])
df['dayofweek'] = df['date'].dt.dayofweek
df['month'] = df['date'].dt.month

# 分析周效应
weekly_avg = df.groupby('dayofweek')[target_col].mean()
print("各星期销量均值:\n", weekly_avg)

# ── Step 5: 关键结论输出 ──
conclusions = {
    '样本量': len(df),
    '时间跨度': f"{df['date'].min()} 至 {df['date'].max()}",
    'SKU数量': df['sku_id'].nunique(),
    '问题类型': '时序回归（多SKU）',
    '算法方向': 'Prophet + LightGBM 集成'
}
for k, v in conclusions.items():
    print(f"{k}: {v}")
