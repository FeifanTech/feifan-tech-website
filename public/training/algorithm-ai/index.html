为什么工程师需要算法思维 + AI协同能力

课程设计理念
本课程不是纯理论的算法教材，而是以飞凡科技真实业务场景为载体，教会工程师如何在实际项目中做出正确的算法选型决策，并借助 AI 工具将开发效率提升至新高度。
🎯
目标导向

从业务问题出发，而非从算法出发。先有问题，再找算法。

🔬
实战驱动

每个知识点对应飞凡实际业务案例，代码可直接复用。

🤖
AI赋能

将 AI 工具融入算法开发全流程，重点讲授 Prompt 工程。

📋 适合人群
✅ 最适合
  • • 有1年以上开发经验的后端/全栈工程师
  • • 已接触过机器学习但缺乏系统方法论
  • • 希望提升算法选型与落地能力
  • • 正在或即将承担算法相关业务需求
⚠️ 前置要求
  • • Python 基础语法（变量/函数/类）
  • • 了解 pandas/numpy 基本用法
  • • 熟悉 git 基本操作
  • • 了解 REST API 基本概念

课程学习目标
完成课程后你将掌握的3项核心能力
🔍

① 业务场景分析能力

能够独立完成从业务需求到算法问题的转化，掌握系统化的分析框架，准确定义目标函数、数据需求和约束条件。

  • 掌握三步业务分析框架（目标→约束→数据）
  • 能正确识别7类算法问题类型
  • 能撰写标准的算法需求文档
⚖️

② 算法选择与实施能力

建立系统的算法选型框架，能够根据具体业务场景的数据规模、精度要求和资源约束，做出合理的算法选型决策。

  • 掌握5维度算法选择框架
  • 熟悉传统ML、深度学习、强化学习的适用边界
  • 能独立完成5个核心业务场景的算法实现
🤖

③ AI协同开发能力

将AI工具融入算法开发全流程，包括数据分析、特征工程、模型调优和代码优化，至少将相关工作效率提升1倍以上。

  • 掌握6类算法开发场景的Prompt模板
  • 能用AI辅助完成数据探索和特征工程
  • 掌握AI辅助超参调优的完整流程
