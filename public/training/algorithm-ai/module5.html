算法项目的特殊性
算法项目不同于普通软件开发——存在大量不确定性（数据质量未知、算法效果难以预测）。项目管理需要留出充足的探索时间，设置合理的里程碑决策点（继续/暂停/转向），并提前规划降级方案（算法不达标时用规则兜底）。
✅ 算法项目管理要点
  • • 前期做好数据可行性评估（3天探索数据比1周建模更值钱）
  • • 设置明确的停止条件（数据不满足时及时止损）
  • • 所有实验用MLflow/wandb记录，可复现
  • • 和业务方保持每周同步（避免技术与业务脱节）
  • • 上线前必须有降级方案
⚠️ 常见项目风险
  • • 数据准备低估工时（通常占全程40%）
  • • 线下好线上差（数据泄露/分布偏移）
  • • 没有建立模型监控就上线
  • • 算法方案无法在生产环境复现
  • • 忽略模型推理延迟和内存占用

需求分析与范围定义
标准化需求文档让算法项目有据可依
📄 算法需求文档模板（PRD for ML）
章节内容要点负责人
1. 背景与目标 业务背景、现状痛点、期望改善的量化指标（如：缺货率从15%降至8%） 产品/业务
2. 功能范围 预测对象（用户/SKU/时间粒度）、输出形式（概率/绝对值/排序）、接入方式 算法+产品
3. 技术指标 主要技术指标及目标值、评估方法（离线/在线）、数据集划分方式 算法
4. 数据需求 数据源清单、字段定义、历史深度要求、更新频率、标注方案（如需） 数据
5. 工程约束 推理延迟上限、模型大小限制、部署环境（CPU/GPU/边缘）、更新频率 工程
6. 里程碑 数据就绪日期、基线完成日期、算法上线日期、效果评估日期 PM
7. 降级方案 若算法不达标或出现故障，降级到什么规则/人工方案 算法+产品
8. 验收标准 算法上线的验收条件（技术指标+业务指标+稳定性测试） 所有相关方

回归/时序预测指标

平均绝对误差。最直观，与目标量纲一致，受异常值影响小。

均方根误差。对大误差惩罚更重，适合不能容忍极端误差的场景。
强调极值
MAPE
平均绝对百分比误差。相对误差，方便跨SKU比较，但零值时分母为0。
销量预测首选
SMAPE
对称MAPE，解决了MAPE在零附近不对称的问题。
有零值场景
🏷️
分类预测指标
AUC-ROC
衡量模型整体区分能力，不受阈值影响，适合不均衡数据。
模型比较
Precision
预测为正例中真正例的比例。关注误报代价高的场景（如：不要骚扰无意愿用户）。
误报代价高
Recall
真正例中被预测为正例的比例。关注漏报代价高的场景（如：不要漏掉潜在买家）。
漏报代价高
F1 Score
Precision和Recall的调和均值，两者兼顾时使用。
均衡场景
Lift
相对于随机策略的提升倍数。Lift@Top10%=3.0意味着top10%用户购买率是随机的3倍。

推荐系统指标
Hit Rate@K
Top-K推荐结果中，命中用户实际购买商品的比例。
离线评估

归一化折扣累积增益，考虑推荐位置，越靠前的命中贡献越大。

三种部署模式对比与飞凡适用场景

📦 批量离线推理

✅ 适用：销量预测、营销打分
⚠️ 不适用：实时个性化推荐
  • • 定时任务（Cron/Airflow）
  • • 结果写入MySQL/Redis
  • • 工程复杂度：低
  • • 实时性：延迟数小时

⚡ 实时在线推理

✅ 适用：实时推荐、购买意愿实时打分
⚠️ 挑战：延迟、高并发、模型热更新
  • • FastAPI/Flask封装推理服务
  • • ONNX/TensorRT模型加速
  • • 工程复杂度：高
  • • 实时性：毫秒级

📡 边缘端部署

✅ 适用：IoT控制、水产监控摄像头
⚠️ 挑战：资源受限、模型压缩
  • • ONNX Runtime轻量推理
  • • 模型量化（INT8）压缩
  • • 工程复杂度：较高
  • • 实时性：超低延迟

FastAPI在线推理服务示例

Python LightGBM模型在线推理服务（FastAPI） 
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import lightgbm as lgb
import numpy as np
import joblib
from typing import List

app = FastAPI(title="购买意愿预测服务")

# 启动时加载模型（单次加载，避免每次推理重新加载）
@app.on_event("startup")
async def load_model():
    app.state.model = joblib.load("models/purchase_intent_v2.pkl")
    app.state.feature_names = joblib.load("models/feature_names.pkl")
    print("模型加载完成")

class PredictRequest(BaseModel):
    user_id: str
    sku_id: str
    features: List[float]

class PredictResponse(BaseModel):
    user_id: str
    sku_id: str
    buy_probability: float
    strategy: str

@app.post("/predict", response_model=PredictResponse)
async def predict(request: PredictRequest):
    try:
        features = np.array(request.features).reshape(1, -1)
        prob = app.state.model.predict_proba(features)[0][1]

        # 业务策略分层
        if prob >= 0.7:   strategy = "优惠券"
        elif prob >= 0.4: strategy = "内容触达"
        elif prob >= 0.2: strategy = "品牌曝光"
        else:             strategy = "不触达"

        return PredictResponse(
            user_id=request.user_id,
            sku_id=request.sku_id,
            buy_probability=round(float(prob), 4),
            strategy=strategy
        )
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

@app.get("/health")
async def health_check():
    return {"status": "ok", "model_loaded": hasattr(app.state, 'model')}

数据漂移检测

Python 特征分布漂移检测（PSI — 群体稳定性指标） 
import numpy as np
import pandas as pd

def calculate_psi(expected: np.ndarray, actual: np.ndarray, buckets: int = 10) -> float:
    """
    计算PSI（Population Stability Index）
    PSI < 0.1: 无明显漂移
    PSI 0.1~0.2: 轻微漂移，建议关注
    PSI > 0.2: 严重漂移，需要重训模型
    """
    breakpoints = np.percentile(expected, np.linspace(0, 100, buckets + 1))
    breakpoints = np.unique(breakpoints)

    expected_pct = np.histogram(expected, bins=breakpoints)[0] / len(expected)
    actual_pct   = np.histogram(actual, bins=breakpoints)[0] / len(actual)

    # 避免除零
    expected_pct = np.where(expected_pct == 0, 0.0001, expected_pct)
    actual_pct   = np.where(actual_pct == 0, 0.0001, actual_pct)

    psi = np.sum((actual_pct - expected_pct) * np.log(actual_pct / expected_pct))
    return psi

# 对所有特征批量检测漂移
def drift_report(train_df: pd.DataFrame, prod_df: pd.DataFrame) -> pd.DataFrame:
    results = []
    for col in train_df.select_dtypes(include=['number']).columns:
        psi = calculate_psi(train_df[col].dropna(), prod_df[col].dropna())
        status = '⚠️漂移' if psi > 0.2 else ('🔍关注' if psi > 0.1 else '✅正常')
        results.append({'特征': col, 'PSI': round(psi, 4), '状态': status})
    return pd.DataFrame(results).sort_values('PSI', ascending=False)

A/B测试最佳实践

🧪 A/B测试设计要点
  • • 实验前用功效分析计算最小样本量
  • • 用户维度随机分组（避免跨会话污染）
  • • 新模型先5%流量，稳定后扩至50%
  • • 运行至少7天（覆盖完整周效应）
  • • 同时看主指标和护栏指标
📊 统计显著性检验
  • • 比例指标（CTR）→ Z检验
  • • 连续指标（GMV/时长）→ T检验
  • • 显著性水平 α=0.05，功效 β=0.80
  • • 避免"偷看"：实验结束前不做决策
  • • 记录实验全程日志，便于事后复盘
