7.1 AI 系统四维观测模型
技术 · 成本 · 质量 · 安全四维指标体系
📊 技术指标
  • 延迟（P50 / P95 / P99）
  • 错误率
  • 吞吐量
💰 成本指标
  • Token 用量
  • 每请求成本
  • 每用户成本
🎯 质量指标
  • 用户满意度
  • 回答相关性
  • 幻觉率
🛡️ 安全指标
  • Prompt 注入尝试
  • 越权访问
  • 内容违规
💡 核心理念：AI 应用的可观测性不仅是技术指标，还包括成本、质量和安全三个 AI 特有维度。四维观测缺一不可。

7.2 LangFuse 集成
全链路追踪与用户评分收集
# ✅ LangFuse 全链路追踪
from langfuse import Langfuse
from langfuse.decorators import observe, langfuse_context

langfuse = Langfuse(
    public_key=os.getenv("LANGFUSE_PUBLIC_KEY"),
    secret_key=os.getenv("LANGFUSE_SECRET_KEY"),
    host=os.getenv("LANGFUSE_HOST", "https://cloud.langfuse.com"),
)

@observe(name="rag_pipeline")  # 自动记录输入/输出/延迟
async def rag_pipeline(query: str, user_id: str) -> str:
    # 追踪检索步骤
    with langfuse.trace(name="retrieval") as span:
        docs = await retrieve_documents(query)
        span.update(
            output={"doc_count": len(docs)},
            metadata={"query": query},
        )
    
    # 追踪生成步骤
    langfuse_context.update_current_observation(
        input={"query": query, "context_length": sum(len(d) for d in docs)},
        user_id=user_id,
    )
    
    answer = await generate_answer(query, docs)
    return answer

# 用户评分收集（用于质量监控）
async def collect_feedback(trace_id: str, score: float, comment: str):
    langfuse.score(
        trace_id=trace_id,
        name="user_satisfaction",
        value=score,            # 0.0 ~ 1.0
        comment=comment,
    )
💡
LangFuse 的核心价值

LangFuse 提供了 LLM 应用的"X-Ray"能力 — 每次请求的输入/输出/延迟/Token 用量/用户评分一目了然。结合用户评分数据，可以持续优化 Prompt 和 RAG 策略。

7.3 Prometheus 指标设计
自定义 AI 指标与监控面板
# ✅ 自定义 AI 指标（Python Prometheus Client）
from prometheus_client import Counter, Histogram, Gauge

# 请求计数
llm_requests_total = Counter(
    'llm_requests_total',
    'Total LLM API requests',
    ['model', 'endpoint', 'status'],
)

# 延迟分布
llm_latency_seconds = Histogram(
    'llm_latency_seconds',
    'LLM request latency',
    ['model'],
    buckets=[0.5, 1.0, 2.0, 5.0, 10.0, 30.0, 60.0],
)

# Token 用量
llm_tokens_total = Counter(
    'llm_tokens_total',
    'Total tokens consumed',
    ['model', 'type'],  # type: input | output
)

# 成本估算（实时）
llm_cost_usd = Counter(
    'llm_cost_usd_total',
    'Estimated LLM cost in USD',
    ['model'],
)

# 当前等待中的请求数
llm_pending_requests = Gauge(
    'llm_pending_requests',
    'Currently in-flight LLM requests',
    ['model'],
)

7.4 告警规则配置
Prometheus 告警规则 — 错误率、延迟与成本异常
# ✅ Prometheus 告警规则
groups:
  - name: llm_alerts
    rules:
      - alert: LLMHighErrorRate
        expr: |
          rate(llm_requests_total{status="error"}[5m])
          / rate(llm_requests_total[5m]) > 0.05
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "LLM 错误率过高 ({{ $value | humanizePercentage }})"
          
      - alert: LLMHighLatency
        expr: |
          histogram_quantile(0.95, rate(llm_latency_seconds_bucket[5m])) > 15
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "LLM P95 延迟超过 15 秒"

      - alert: LLMCostAnomaly
        expr: |
          increase(llm_cost_usd_total[1h]) > 50
        labels:
          severity: warning
        annotations:
          summary: "过去 1 小时 LLM 成本超过 $50"
⚠️ 重要：成本告警是 AI 应用特有的需求——一个失控的循环或意外的流量激增可以在几小时内产生数千美元的费用。始终设置成本硬上限和告警。
