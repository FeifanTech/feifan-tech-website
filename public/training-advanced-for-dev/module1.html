选型原则

每一层都应该可独立替换。不要让具体 LLM 厂商 API 侵入业务逻辑——通过 LLM Gateway 统一抽象，确保模型层可随时切换。

1.3 关键设计决策
三大核心原则 + 四大反模式
三大核心设计原则

🔒 原则一：语境隔离原则

每个用户会话维护独立的上下文窗口，避免跨会话污染。上下文超限时采用滚动摘要而非截断，保留关键信息连贯性。

✅ 原则二：确定性优先原则

业务关键路径不依赖 LLM 输出，AI 仅用于辅助决策。所有 LLM 输出必须经过结构化解析和 Schema 验证，确保下游系统接收到确定性数据。

🔄 原则三：优雅降级原则

主模型不可用时自动切换备用模型；AI 功能不可用时回退到规则引擎。降级策略需纳入 SLA 承诺范围，确保业务连续性。

四大反模式与修正方案
❌ 反模式 ✅ 修正方案
同步等待超长 LLM 调用 改为流式响应 + 乐观 UI
将 Prompt 硬编码于代码 迁移至 Prompt Registry
忽略 Token 用量统计 接入 Cost Tracking
LLM 直接访问生产数据库 通过工具函数代理访问
⚠️ 特别警示：让 LLM 直接生成并执行 SQL 查询生产数据库是最危险的反模式之一。务必通过经过鉴权和参数校验的工具函数进行间接访问。

路由配置示例（YAML）
# 多模型路由配置 routes: - name: simple_qa matcher: token_budget: "<500" intent: ["greeting", "faq", "lookup"] primary: claude-haiku-3-5 fallback: gpt-4o-mini timeout_ms: 5000 - name: complex_reasoning matcher: token_budget: ">=500" intent: ["analysis", "coding", "planning"] primary: claude-3-7-sonnet fallback: gpt-4o timeout_ms: 30000 cost_controls: daily_budget_usd: 500 alert_threshold: 0.8 hard_cap: true
💡
成本控制要点

始终设置 daily_budget_usd 硬上限和 alert_threshold 预警阈值。80% 的请求通常可以路由到低成本模型，仅将 20% 的复杂任务分配给高端模型，从而在质量和成本之间取得最佳平衡。
