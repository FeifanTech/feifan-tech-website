4.1 Agent 执行模型
ReAct（Reasoning + Acting）循环架构
ReAct 循环流程
📋 Task
用户任务输入
→
💭 Thought
LLM 推理
→
⚡ Action
调用工具
→
👁️ Observation
工具返回结果
→
🔄 循环
回到 Thought
ReAct 循环三要素
要素 角色 说明
Thought 推理 LLM 推理当前状态和下一步计划
Action 执行 调用工具（函数）
Observation 观察 工具返回结果，注入上下文
💡 核心理念：Agent = LLM + 工具 + 循环。LLM 负责推理和决策，工具负责执行，循环直至任务完成或达到最大步数。

4.2 工具函数设计规范
标准工具函数定义与 Schema 设计
工具函数示例（Python + Pydantic）
import json
from typing import Annotated
from pydantic import BaseModel, Field

class SearchParams(BaseModel):
    query: Annotated[str, Field(description="搜索关键词，支持自然语言")]
    max_results: Annotated[int, Field(default=5, ge=1, le=20, description="最多返回结果数")]
    date_range: Annotated[str | None, Field(
        default=None, 
        description="日期范围，格式: YYYY-MM-DD/YYYY-MM-DD"
    )]

async def search_knowledge_base(params: SearchParams) -> dict:
    """
    在企业知识库中搜索相关文档。
    
    返回格式：
    {"results": [{"title": str, "content": str, "score": float}], "total": int}
    """
    try:
        results = await kb_retriever.search(
            query=params.query,
            limit=params.max_results,
            date_filter=params.date_range,
        )
        return {
            "results": [r.to_dict() for r in results],
            "total": len(results),
        }
    except Exception as e:
        # ⚠️ 工具必须处理异常，不能让错误传播到 Agent 循环外
        return {"error": str(e), "results": [], "total": 0}
💡
工具函数设计三原则

① 参数必须有明确的类型和描述（LLM 靠描述理解工具用途）② 返回值结构固定（Agent 需要稳定的解析）③ 内部异常必须捕获（防止循环崩溃）

4.3 工具安全设计
五大安全风险与防御措施
五大安全风险对照表
风险类型 场景 防御措施
命令注入 工具执行 shell 命令 参数白名单 + 沙箱环境
路径遍历 工具读取文件 路径前缀校验
权限越界 工具访问数据库 只读账号 + 行级权限
资源滥用 工具发起外部请求 域名白名单 + 速率限制
数据泄露 工具返回敏感字段 输出脱敏处理
⚠️ 特别警示：永远不要让 Agent 拥有未经审核的写权限。所有修改数据的操作（DELETE、UPDATE、POST 到外部API）必须经过人工确认（HITL）机制。

workflow = StateGraph(AgentState)

4.5 Agent 稳定性保障
五大稳定性措施确保 Agent 可靠运行
五大稳定性措施
1
超时控制

每个工具调用设置独立超时（建议 10-30s）

2
重试策略

指数退避重试（最多 3 次，避免无效循环）

3
最大步数限制

防止 Agent 陷入无限循环（建议 max_steps=15）

4
检查点（Checkpoint）

长任务每步持久化状态，支持断点续跑

5
人机协作（HITL）

高风险操作（写操作/外部调用）需人工确认

💡
生产环境安全阀

生产环境中，Agent 最大步数和超时是最重要的两个安全阀。一个失控的 Agent 循环可以在几分钟内消耗数百美元的 Token。
