2.1 Prompt 结构框架（RISEN）
五要素结构化 Prompt 设计方法论
RISEN 框架要素一览
要素 英文 作用 示例
角色 Role 定义模型身份 你是一位资深 Java 架构师
指令 Instruction 明确任务目标 分析以下代码的性能瓶颈
场景 Scenario 提供业务背景 用于高并发电商订单系统
期望 Expectation 定义输出格式 以 JSON 格式输出，包含问题列表和优化建议
约束 Negative 排除不需要的内容 不要包含与 Java 无关的建议
💡 核心理念：RISEN 框架的每个要素都在缩小 LLM 的输出空间，要素越完整，输出越精准。缺失任何一个要素都可能导致输出偏离预期。

2.2 System Prompt 设计规范
高质量 System Prompt 模板与最佳实践
高质量 System Prompt 模板
## 身份定义
你是 [公司名] 的 AI 助手，专注于 [领域]。

## 能力边界
- 可以：[列出允许的操作]
- 不可以：[列出禁止的操作]

## 输出格式
所有回复必须遵循以下结构：
1. 直接回答（≤2句话）
2. 详细说明（如需要）
3. 操作建议（如适用）

## 质量标准
- 使用简洁的技术语言
- 代码示例必须包含注释
- 不确定时明确说明
💡
设计要点

System Prompt 是 LLM 应用的"宪法"——它定义了 AI 的行为边界和输出标准。好的 System Prompt 应该具备：身份明确、边界清晰、格式统一、标准可衡量。

2.3 高级 Prompt 技巧
链式思维、少样本学习与结构化输出控制
1
链式思维（Chain-of-Thought）
引导模型逐步推理，显著提升复杂任务准确率

通过要求模型在给出最终答案前展示推理过程，可以有效减少"幻觉"和逻辑错误。使用 XML 标签分隔思考过程和最终答案。

import openai

response = openai.chat.completions.create(
    model="gpt-4o",
    messages=[
        {
            "role": "system",
            "content": (
                "你是一位资深代码审查专家。\n"
                "分析问题时，请先在 <thinking> 标签内逐步推理，\n"
                "然后在 <answer> 标签内给出最终结论。"
            )
        },
        {
            "role": "user",
            "content": (
                "以下 Python 函数在高并发场景下是否线程安全？\n"
                "请逐步分析潜在的竞态条件。\n\n"
                "def transfer(from_acc, to_acc, amount):\n"
                "    if from_acc.balance >= amount:\n"
                "        from_acc.balance -= amount\n"
                "        to_acc.balance += amount\n"
                "        return True\n"
                "    return False"
            )
        }
    ]
)

# 解析 <thinking> 和 <answer> 标签内容
result = response.choices[0].message.content
print(result)
2
少样本学习（Few-Shot）
通过示例引导模型输出特定格式和风格

提供 1–3 个输入-输出示例，让模型学习期望的格式和逻辑模式，比纯文字描述更精准高效。

import openai

few_shot_messages = [
    {
        "role": "system",
        "content": "你是一个代码审查助手，按照固定格式输出审查结果。"
    },
    # 示例 1
    {
        "role": "user",
        "content": "审查：password = request.args.get('pwd')"
    },
    {
        "role": "assistant",
        "content": (
            "🔴 严重度: HIGH\n"
            "📂 类别: 安全漏洞\n"
            "💬 说明: 密码通过 URL 参数传输，存在泄露风险\n"
            "✅ 建议: 改用 POST 请求体传输，并启用 HTTPS"
        )
    },
    # 示例 2
    {
        "role": "user",
        "content": "审查：time.sleep(30)"
    },
    {
        "role": "assistant",
        "content": (
            "🟡 严重度: MEDIUM\n"
            "📂 类别: 性能问题\n"
            "💬 说明: 硬编码长时间阻塞，影响系统吞吐量\n"
            "✅ 建议: 使用异步等待或可配置的超时参数"
        )
    },
    # 实际请求
    {
        "role": "user",
        "content": "审查：eval(user_input)"
    }
]

response = openai.chat.completions.create(
    model="gpt-4o",
    messages=few_shot_messages
)
print(response.choices[0].message.content)
3
结构化输出控制
使用 JSON Schema 约束模型输出格式，确保下游系统可解析

通过 response_format 参数传入 JSON Schema，强制模型输出符合预定义结构的数据，消除格式不确定性。

import openai

response = openai.chat.completions.create(
    model="gpt-4o",
    messages=[
        {
            "role": "system",
            "content": "你是一个代码质量分析助手，严格按照指定 JSON 格式输出。"
        },
        {
            "role": "user",
            "content": "分析以下代码的质量问题：\nfor i in range(len(lst)): print(lst[i])"
        }
    ],
    response_format={
        "type": "json_schema",
        "json_schema": {
            "name": "code_review",
            "schema": {
                "type": "object",
                "properties": {
                    "issues": {
                        "type": "array",
                        "items": {
                            "type": "object",
                            "properties": {
                                "severity": {
                                    "type": "string",
                                    "enum": ["HIGH", "MEDIUM", "LOW"]
                                },
                                "category": { "type": "string" },
                                "description": { "type": "string" },
                                "suggestion": { "type": "string" }
                            },
                            "required": ["severity", "category",
                                         "description", "suggestion"]
                        }
                    },
                    "overall_score": {
                        "type": "number",
                        "minimum": 0,
                        "maximum": 10
                    }
                },
                "required": ["issues", "overall_score"]
            }
        }
    }
)
print(response.choices[0].message.content)
